Grand Rounds Data Challenge

The federal government recently began publicly disclosing provider-identifiable Medicare outpatient data (cf. CMS). We have provided some cleaned versions of two such data sets and some background information for you to use in the solution of a problem we will present below.

Data

  1. Medicare Procedure Claims: The first data set is contained in npi_proc_counts.csv and holds records from 2013 indicating how many times each physician billed Medicare for a specific procedure code (these are "HCPCS" or "CPT" codes) . Procedure codes exist for every type of surgery, test, office visit, and office-administered treatment. The columns are:
    1. NPI: This is the NPI ("National Provider Identifier) of the given provider;
    2. HCPCS_CODE: the procedure code of the service;
    3. LINE_SRVC_CNT: the number of times the service was performed;
    4. BENE_UNIQUE_CNT: the number of unique Medicare beneficiaries receiving the service from the given provider.
  2. Medicare Pharmacy Claims :The second data set, contained in npi_drugs.csv contains records from 2013 indicating how many prescriptions each physician billed Medicare for a specific drug. The columns are:
    1. NPI: Same as in npi_proc_counts;
    2. DRUG_NAME: the brand name of the drug in question;
    3. GENERIC_NAME: the generic name of the drug in question;
    4. TOTAL_CLAIM_COUNT: the total number of claims for the drug associated to the given provider.

In addition, there are some auxillary files that contain other helpful information :

  1. hcpcs_lookup.csv contains a lookup table for the HCPCS codes that show up in npi_proc_counts.csv. This will allow you to associate these codes with human-readable descriptions. The columns are:
    1. HCPCS_CODE: The HCPCS code in question;
    2. HCPCS_DESCRIPTION: a readable description of the code.
  2. npi_specialty.csv contains a list of pairs (npi, specialty) which describes the given specialty of the physician referenced by the NPI. The columns are:
    1. NPI: The NPI of the given physician;
    2. specialty: the specialty of the given physician.

Problem Statement

You should aim to spend approximately 2-3 hours on this challenge. We'd like to mostly get a sense of how you approach problem solving and data analysis, so do not feel obligated to have produced a complete, perfect analysis in this short time span. You are welcome (and encouraged) to include a discussion of how you would proceed given more time.

Main Goal

The primary objective of this exercise is as follows: There is a collection of doctors in npi_specialty.csv that are labelled as cardiologists, and we would like you to produce a generalizable method for distinguishing these doctors from the general population.

That is:

  • If we were to give you the same data for another year, you ought to be able to give us a list of cardiologists using the same methodology (indeed, CMS has other years of similar Medicare data if you are interested in doing this).
  • This method should not rely on any external data that amounts to "hard-coding" the fact that the given doctors are cardiologists.
  • Ideally, there would be some aspects of this method that can be leveraged to provide some usable insights for how cardiologists behave relative to other specialties, but this is not a requirement.

Secondary Goals

In addition to the main problem, we would like you to discuss your analysis. Some things you might decide to talk about (but are not limited to) are:

  • Why/how did you choose the classification procedure you decided on?
  • How is this generalizable?
  • What other types of data did you use/consider using in your analysis? What other data would you use (and how) if it were freely available to you?
  • If it is possible, are there interpretable components of your model that can be used for an understanding of how cardiologists are distinct from the general population of doctors?

Deliverables

Please provide:

  1. Your documented code used in the analysis;
  2. Some discussion of your results as described above.

These may be contained in the same document, and you may use whatever programming language / packages / document format you like.
